Routes

A system called routing in Rails, matches HTTP requests from browser to the app

Request from browser

Requests made from browser (by typing in a URL) would have a HTTP method defined so that the app knows what actions to perform

Routes in Rails app

This request is received by the routing system and refers to 

config/routes.rb
file. The routing system looks for:

  • an associated controller eg: 
    ideas_controller.rb
  • an action(function) in the controller eg: 
    def index end;
  • parameters if any eg: 
    /ideas/:id

Render views

Determines which page to show on browser 

ideas/index.html.erb

Recap

  • URL entered in browser will be checked if a corresponding entry exists in 
    config/routes.rb
  • If there is a match in the routes, the request goes to corresponding action in the controller
  • Action renders a corresponding view

HTTP methods

There are 4 HTTP methods:

  • GET
  • PUT
  • POST
  • DELETE

What is a resource?

Resource is something that can be created, read, updated, deleted CRUD

Idea in this app is modelled as a resource that can be created, read, updated and deleted

Matching HTTP method to a resource

A HTTP method is matched to a resourece

GET /ideas/new render a form to create a new idea

POST /ideas will create a new idea and saves in the database

GET /ideas/1 display a specific idea

Rails Generator

Rails scaffold generates routes for each resource we create in config/routes.rb

resources :ideas

7 different routes for each resource

HTTP Method Path Controller Action used for
GET /ideas index displaying a list of all ideas
GET /ideas/new new returning an HTML form for creating a new idea
POST /ideas create creating a new idea
GET /ideas/:id show displaying a specific idea
GET /ideas/:id/edit edit returning an HTML form for editing a specific idea
PUT /ideas/:id update updating a specific idea
DELETE /ideas/:id destroy deleting a specific idea

Recap

  • In Rails, concepts are modeled as a resource whereby it can be created, read, updated, deleted CRUD
  • A HTTP methods are matched to a resource. 
    GET /ideas/new
    returns a HTML page with a form to create a new idea

Controller

  • Rails executes a corresponding controller
    • class IdeasController < ApplicationController
  • Every controller contains a few actions , eg: 
    def index; end
    is an action that queries data from database and passes it on to the view

Inside a controller

  • Sets an instance variable denoted by an @. Idea is an ActiveRecord model that connects to the database
    • @ideas = Idea.all
  • Renders a html file in a view
    • 
        respond_to do |format|
          format.html
          format.json { render json: @idea }
        end
    points to a html file with the same name,
    app/views/ideas/index.html.erb
